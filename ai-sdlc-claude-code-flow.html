3 Implementer setup — Claude Code

Prerequisites

You need: An Anthropic API key, an ADO project with a PAT (Work Items RW, Code RW, Build RW), a Slack incoming webhook URL, and claude CLI installed locally.
1
Copy the starter into your repo
copy ai-sdlc-orchestrator <your-repo>
This puts .claude/, .azure-pipelines/, and docs/ in place.
2
Run bootstrap
cd <your-repo> && claude -p "/bootstrap-project"
Scans the repo, generates .claude/project/PROFILE.draft.md with cited evidence.
3
Review and activate the profile
Open PROFILE.draft.md. Fix every [NEEDS HUMAN INPUT] entry. Confirm stack detection is correct.
mv .claude/project/PROFILE.draft.md .claude/project/PROFILE.md
4
Fill in guideline stubs
Edit files in .claude/project/guidelines/ — coding standards, error handling, naming, API contracts. Add project-specific overrides in .claude/project/overrides/.
5
Create ADO variable group
Project Settings → Library → New variable group ai-sdlc-secrets:
ANTHROPIC_API_KEY (secret), ADO_PAT (secret), SLACK_WEBHOOK_URL (secret).
6
Create four pipelines
Each pointing at its YAML in .azure-pipelines/:
design-gen, implement-story, pr-review, post-merge.
7
Wire the ADO Service Hook
Project Settings → Service Hooks → Work Item Updated → Filter: tags contain ai:ready → Action: trigger design-gen pipeline, mapping workItemId.
8
Set branch policies
On main and design/**: require 1 human reviewer, require PR build to pass, prevent direct pushes. This is the merge safety net.
9
Commit and push
git add .claude/ .azure-pipelines/ docs/ && git commit -m "Adopt AI-SDLC" && git push
10
Tag your first work item
Write a clear description in an ADO Feature, tag it ai:ready, and watch the design PR appear within a few minutes.
First-run expectation: The design PR will likely need 1–2 revision comments from the tech lead. The slicing will probably be too fine. That's normal — correct in the PR, and the system learns from the overrides you add.
